# 本地项目管理实现说明

- 状态：真实项目管理、Git 文件变更摘要与自动刷新已通过 Windows Preview
- 最后更新：2026-06-15
- 对应目录：`apps/desktop/src/projects.ts`、`apps/desktop/src-tauri/src/main.rs`
- 首次项目管理权威构建：[Windows Preview #27334790840](https://github.com/MorganArthur/mimodex/actions/runs/27334790840)

## 1. 本阶段目标

本阶段把桌面壳中的硬编码项目替换为真实本地项目记录，使用户可以选择本地文件夹、
查看 Git 摘要、切换当前项目，并让新任务以所选项目路径作为 Runtime 工作目录。

## 2. 已实现范围

- 使用 Tauri 原生文件夹选择器添加本地项目；
- 在应用数据目录的 `projects.json` 中持久化非敏感项目摘要；
- 启动时恢复项目列表与上次选中的项目；
- 读取项目是否可用、Git 分支、短提交号和工作区变更数量；
- 聚合已暂存、未暂存和未跟踪文件变更，并展示各文件新增与删除行数；
- 页面可见且任务空闲时每 `30` 秒后台刷新 Git，任务结束后立即刷新；
- 项目列表保持固定顺序；切换、刷新和 Git 状态变化只更新内容，不改变已有项目位置；
- 新添加的项目进入列表顶部；
- 切换项目时先更新本地界面，再由后台持久化选择状态；下一次任务使用新项目路径创建 Runtime 线程；
- 项目文件夹被移动或删除时显示不可用状态，并禁止启动任务；
- 浏览器开发环境继续使用可替换的演示项目服务。

项目记录包含：

| 字段 | 用途 |
| --- | --- |
| `id` | 规范化本地路径生成的稳定项目标识 |
| `path` | Runtime 创建线程时使用的工作目录 |
| `name` | 侧栏显示名称 |
| `available` | 文件夹当前是否可访问 |
| `git` | Git 仓库、分支、HEAD 与变更数量摘要 |
| `lastOpenedAt` | 兼容已有项目记录的时间字段；不再用于列表重排 |

API Key 不进入项目记录，仍只保存在 Windows 凭据管理器。

## 3. 安全与边界

Git 状态检查只执行固定的只读命令：

```text
git -C <project> rev-parse --show-toplevel
git -C <project> branch --show-current
git -C <project> rev-parse --short HEAD
git -C <project> status --porcelain=v1 --untracked-files=normal
```

项目路径作为独立进程参数传入，不拼接为 shell 命令。项目列表加载不会递归读取项目
文件，也不会修改 Git 元数据。

## 4. 当前限制

- 尚未提供“从 Mimodex 移除项目”；该 P1 功能只能移除记录，不能删除本地文件；
- Git 状态仍通过固定短命令读取，已移入 Tauri 后台任务并串行保护持久化写入；后续应增加原生超时；
- `projects.json` 仅保存项目摘要；线程历史已由独立 SQLite 事件账本持久化；
- 线程归档已实现，项目分组、线程重命名和置顶仍未实现；
- 项目显示名称当前取文件夹名，尚未提供自定义名称。
