# Mimodex 产品需求文档

- 状态：待评审草案
- 版本：0.1
- 最后更新：2026-06-09
- 目标版本：私有 Beta
- 首要平台：Windows
- 产品语言：中文优先，预留英文国际化能力
- 外部 API 基线核实日期：2026-06-09

## 1. 产品概述

Mimodex 是一款由 MiMo v2.5 与 MiMo v2.5 Pro 驱动的本地优先桌面编程
Agent。它通过读取代码、制定方案、编辑文件、运行命令、审阅 Diff 和与用户持续
协作，帮助开发者完成限定在本地代码仓库内的工程任务。

Mimodex 不是通用聊天客户端，也不以替代完整 IDE 为目标。它的核心价值是：
在开发者的本地项目中提供可信、可审阅、可控制的 Agent 执行闭环。

## 2. 要解决的问题

MiMo 用户可以直接调用模型 API，但要构建真正有效的编程 Agent，还必须解决：

- 如何高效获取代码仓库上下文；
- 如何在本地执行工具，并将结果以结构化方式返回模型；
- 如何限制高风险操作并获得用户审批；
- 如何在多轮对话中完整保留模型推理和工具调用历史；
- 如何让文件改动与命令输出可被审阅；
- 如何在任务中断后安全恢复工作。

Mimodex 将这些能力整合为一套专注的桌面端工作流。

## 3. 产品目标

### 3.1 私有 Beta 目标

1. 开发者可以在两分钟内添加本地 Git 仓库并启动任务。
2. Agent 可以使用 MiMo 完成常见的仓库内编程任务。
3. 每次文件变更和命令执行都可见，并能追溯到对应线程。
4. 高风险操作会根据当前权限模式被阻止或请求审批。
5. 已停止或已关闭的线程能够恢复相关上下文。
6. 用户可以在接受结果前检查最终 Diff 与验证结果。

### 3.2 成功指标

私有 Beta 的遥测必须默认关闭。即使用户主动开启，也不得默认采集源代码、
提示词、命令输出或文件内容。

| 指标 | Beta 目标 |
| --- | --- |
| 有效本地 Git 仓库添加成功率 | 不低于 95% |
| 首个有效 Agent 动作 | p50 小于 30 秒，不含模型排队时间 |
| 正常持久化线程恢复成功率 | 不低于 99% |
| 工具调用协议失败率 | 低于模型轮次的 2% |
| 无法解释的文件改动 | 0 |
| 桌面端无崩溃会话占比 | 不低于 99% |
| 用户反馈的 Beta 任务完成率 | 不低于 60% |

## 4. 首版不做的内容

私有 Beta 不包含：

- 云端执行或托管代码仓库；
- 定时自动化；
- 浏览器或通用桌面控制；
- 多 Agent 或子 Agent 编排；
- 插件或 MCP 市场；
- IDE 插件同步；
- 自动创建 Pull Request；
- 组织账号、计费和集中式策略管理；
- 内置完整代码编辑器；
- 未经用户选择项目的跨目录自主操作。

这些是首版范围决策，并不代表永远不支持。

## 5. 目标用户

### 5.1 核心用户画像

主要面向在本地代码仓库中工作的开发者，他们：

- 能够审阅代码 Diff 和终端输出；
- 拥有 MiMo API 凭据；
- 希望 Agent 协助实现功能、调试、测试或代码审查；
- 重视本地控制与明确的权限边界。

### 5.2 首个平台用户画像

首个 Beta 面向使用 Windows、PowerShell 和 Git 的开发者。Windows 执行、
打包与沙箱模型稳定后，再支持 macOS。

## 6. 核心用户流程

1. 用户安装 Mimodex 并填写 MiMo API 凭据。
2. 用户添加一个本地项目文件夹。
3. Mimodex 检测仓库状态、分支、工作区改动和本地指导文件。
4. 用户创建线程，选择模型和权限模式，并提交任务。
5. Agent 获取上下文并流式展示活动。
6. Agent 在操作越过当前策略边界前请求审批。
7. Agent 编辑文件并运行验证命令。
8. 用户审阅活动记录、命令结果和 Git Diff。
9. 用户继续补充指令、接受结果，或通过 Git 感知的控制撤销选定改动。
10. 线程被保留，之后可以恢复。

## 7. 产品原则

- 本地优先：项目数据和会话记录默认仅保存在设备本地。
- 天然可审阅：所有动作和修改都必须可检查。
- 最小权限：执行从满足任务所需的最窄权限开始。
- 用户可干预：用户可以中止、批准、拒绝并重定向任务。
- 状态真实：界面明确区分模型输出、计划动作、运行中工具、完成结果、失败和审批。
- Provider 感知：Runtime 正确保留 MiMo 的推理与工具调用语义，而非将其当作普通文本生成接口。

## 8. 功能需求

优先级定义：

- P0：私有 Beta 必须完成；
- P1：Beta 后应尽快完成；
- P2：明确延后。

### 8.1 初始设置

| 编号 | 优先级 | 需求 | 验收标准 |
| --- | --- | --- | --- |
| SET-001 | P0 | 配置 MiMo 凭据 | 可以验证凭据，且不会写入日志或明文配置 |
| SET-002 | P0 | 使用操作系统凭据存储 | 重启后仍可使用，凭据不出现在项目文件中 |
| SET-003 | P0 | 配置 API Base URL | 可选择官方端点或输入兼容的自定义地址 |
| SET-004 | P0 | 选择默认模型 | 默认展示并使用 `mimo-v2.5`；`mimo-v2.5-pro` 仅在高级模型选择中展示 |
| SET-005 | P0 | 配置默认权限模式 | 新线程继承该模式，执行前清晰显示 |
| SET-006 | P1 | 连接诊断 | 分别报告认证、端点、延迟和模型可用性 |
| SET-007 | P1 | 导入导出非敏感设置 | 导出内容不包含凭据和本地会话数据 |

### 8.2 项目管理

| 编号 | 优先级 | 需求 | 验收标准 |
| --- | --- | --- | --- |
| PRJ-001 | P0 | 添加本地文件夹为项目 | 保存路径、显示名称、Git 状态和最近打开时间 |
| PRJ-002 | P0 | 检测 Git 仓库状态 | 展示分支、HEAD、脏工作区和未跟踪文件 |
| PRJ-003 | P0 | 遵守项目边界 | 工作区写入模式下，未经审批不得修改项目外内容 |
| PRJ-004 | P0 | 读取持久项目指导 | 按目录作用域发现并加载适用的 `AGENTS.md` |
| PRJ-005 | P1 | 从 Mimodex 移除项目 | 移除项目记录不会删除项目文件 |
| PRJ-006 | P2 | 托管 Git Worktree | 等本地线程工作流稳定后再实现 |

| 编号 | 优先级 | 需求 | 验收标准 |
| --- | --- | --- | --- |
| THR-001 | P0 | 创建项目绑定线程 | 每个线程记录项目、模型、权限模式和创建时间 |
| THR-002 | P0 | 流式展示 Agent 事件 | 增量展示文本、推理状态、工具、审批和错误 |
| THR-003 | P0 | 停止活动任务 | 停止模型请求，并阻止尚未开始的工具执行 |
| THR-004 | P0 | 恢复线程 | 恢复消息、推理内容、工具调用、结果和审批记录 |
| THR-005 | P0 | 发送后续指令 | 后续轮次使用既有线程状态和当前项目状态 |
| THR-006 | P0 | 显示模型和用量 | 显示当前模型以及可获得的 Token 或用量数据 |
| THR-007 | P1 | 重命名、置顶和归档线程 | 操作不会修改项目文件 |
| THR-008 | P1 | 压缩长上下文 | 压缩后仍保留 MiMo 协议要求的字段 |
| THR-009 | P2 | 从历史分叉线程 | 等恢复可靠性建立后再实现 |

### 8.5 权限与审批

| 编号 | 优先级 | 需求 | 验收标准 |
| --- | --- | --- | --- |
| SEC-001 | P0 | 只读模式 | 文件写入和修改性命令无法执行 |
| SEC-002 | P0 | 工作区写入模式 | 项目内可写；越界操作必须审批 |
| SEC-003 | P0 | 完全访问模式 | 启用前明确警告，启用后始终醒目显示 |
| SEC-004 | P0 | 审批请求界面 | 展示动作、原因、影响边界和可选决策 |
| SEC-005 | P0 | 处理拒绝结果 | 被拒绝的工具不执行，结构化结果返回模型 |
| SEC-006 | P0 | 分离模型网络与命令网络 | 调用 MiMo 不会自动允许 Shell 命令联网 |
| SEC-007 | P0 | 审计记录 | 持久化工具请求、审批决定、执行结果和时间 |

### 8.6 审阅与 Git 体验

| 编号 | 优先级 | 需求 | 验收标准 |
| --- | --- | --- | --- |
| REV-001 | P0 | 展示仓库 Diff | 编辑后更新，并区分新增、修改、删除和未跟踪文件 |
| REV-002 | P0 | 展示单文件 Diff | 提供易读的统一或分栏视图 |
| REV-003 | P0 | 关联动作与变更 | 文件编辑可追溯至工具事件和线程轮次 |
| REV-004 | P0 | 保留用户既有改动 | Mimodex 不得静默撤销用户修改 |
| REV-005 | P1 | 撤销选定 Agent 改动 | 操作可预览、Git 感知，且不覆盖无关用户改动 |
| REV-006 | P1 | 在 Diff 上提交反馈 | 反馈可作为线程上下文发送 |

### 8.7 终端

| 编号 | 优先级 | 需求 | 验收标准 |
| --- | --- | --- | --- |
| TRM-001 | P0 | 展示 Agent 命令输出 | 流式显示命令、工作目录、状态和耗时 |
| TRM-002 | P0 | 取消运行中的命令 | 在平台支持时终止整个进程树 |
| TRM-003 | P1 | 用户控制的集成终端 | 用户可独立于 Agent 运行命令 |
| TRM-004 | P1 | 将当前终端输出作为上下文 | 需要用户明确触发，并区分用户终端和 Agent 工具 |

## 9. 必需界面

### 9.1 项目与线程侧栏

- 项目列表和添加项目入口；
- 按项目分组的线程列表；
- 活动中、运行中和错误状态；
- 已归档线程入口。

### 9.2 新建线程界面

- 任务输入框；
- 当前项目；
- 默认展示 `mimo-v2.5` 的模型选择器，以及可展开的高级模型选择；
- 权限选择器；
- 当前执行边界的清晰说明。

### 9.3 线程工作区

- 流式对话与活动时间线；
- 审批卡片；
- 停止控制；
- 终端与活动面板；
- Git Diff 面板；
- Token 与模型状态。

### 9.4 设置界面

- Provider 凭据与端点；
- 默认模型；
- 默认权限；
- 本地数据路径；
- 连接诊断；
- 隐私和可选遥测设置。

## 10. Agent 事件模型

界面必须渲染结构化事件，不得通过解析纯文本推断状态：

- 用户消息；
- 助手文本增量；
- 推理状态与推理增量；
- 计划更新；
- 工具调用请求；
- 审批请求、批准或拒绝；
- 工具开始、输出增量、完成、失败或取消；
- 文件变更记录；
- 用量更新；
- 轮次完成、失败或取消。

线程恢复后，事件顺序必须保持稳定。

## 11. MiMo 专属要求

1. 对 Provider 返回的每条助手消息保存 `reasoning_content`。
2. 后续请求需要时，完整重放助手推理和工具调用历史。
3. 支持流式文本、推理和工具参数增量。
4. 缺失或格式错误的工具参数必须作为可恢复协议错误处理，绝不能直接作为 Shell 命令执行。
5. 展示 Provider 请求 ID 和归一化错误类别，但不得泄露凭据。
6. 任何模型能力只有在通过 Provider 验证清单后才可宣称支持。

## 12. 非功能需求

### 12.1 安全与隐私

- 凭据不得写入项目文件、会话记录、日志、崩溃报告或遥测。
- 项目内容仅在当前任务需要时发送到配置的 MiMo 端点。
- Agent 命令默认禁止联网。
- 日志必须支持脱敏和有限保留。
- 完全访问不得作为默认模式。

### 12.2 可靠性

- 桌面端重启不得破坏已完成线程记录。
- 崩溃中断的轮次必须恢复为“已中断”，不能错误标记为完成。
- UI 持久化层必须能幂等处理重复事件。
- 工具超时和输出限制必须可配置并可见。

### 12.3 性能

- Beta 目标设备上，应用外壳应在五秒内可交互。
- 打开项目时不得递归读取整个仓库。
- 搜索和工具输出进入模型上下文前必须受限。
- 大量命令输出期间，流式界面仍应保持响应。
- Windows x64 安装包日常优化目标不超过 `100 MiB`，超过 `120 MiB` 不得发布。
- Windows x64 安装后的应用文件总量不得超过 `350 MiB`。
- 安装包不得包含 Rust 工具链、Cargo 缓存、源码、测试产物或未剥离的调试符号。

### 12.4 无障碍与国际化

- 主要操作均可通过键盘完成。
- 状态不能仅依赖颜色表达。
- 所有用户可见文本必须支持国际化。
- Beta 默认语言为中文。

## 13. Beta 发布门槛

满足以下条件后，才能启动私有 Beta：

1. 所有 P0 需求都有负责人和验收测试；
2. MiMo Provider Spike 强制条件全部通过；
3. 凭据脱敏测试通过；
4. Windows 上只读和工作区写入边界验证通过；
5. 中断线程恢复经过测试；
6. 能完成三类代表性仓库任务：小范围 Bug 修复、带测试的功能修改、代码审查；
7. Windows 安装包与安装后文件总量通过体积预算检查；
8. 发布说明记录所有已知破坏性操作风险。

## 14. 代表性验收场景

### 场景 A：小范围 Bug 修复

仓库中存在失败单元测试。用户要求 Mimodex 定位并修复原因。Agent 读取相关文件，
只修改项目内内容，运行聚焦测试，并展示最终 Diff 与测试结果。

### 场景 B：拒绝高风险命令

当前为工作区写入模式，Agent 提议执行需要联网或写入项目外的命令。Mimodex
阻止执行并请求审批；用户拒绝后，将结构化拒绝结果返回 Agent。

### 场景 C：恢复包含工具调用的线程

线程中已包含 MiMo 推理内容和工具调用。用户关闭并重启 Mimodex，之后的跟进请求
不会触发 Provider 协议错误，并能保留原活动时间线。

### 场景 D：保留用户改动

仓库中已有未提交修改，Agent 又修改了其他内容。审阅界面能够区分既有改动与
Agent 活动，Mimodex 不会静默丢弃任何一方的修改。
