## 2. 安装与生命周期

- [ ] 从 GitHub Pre-release 下载安装包和 `.sha256` 文件。
- [ ] 校验安装包 SHA256 与 Release 校验文件一致。
- [ ] 在未安装 Rust、Node.js 的 Windows 11 x64 设备上完成安装。
- [ ] 首次启动正常，无白屏、无不可交互区域。
- [ ] 退出并重新启动后应用正常恢复。
- [ ] 卸载成功，开始菜单和安装目录无异常残留。

## 3. 凭据、端点与诊断

- [ ] 首次设置必须输入 MiMo API Key，空值不能保存。
- [ ] 正确 Key 可通过诊断并保存到 Windows 凭据管理器。
- [ ] 错误 Key 显示“认证失败”，且不会被保存。
- [ ] 官方端点可诊断成功，并显示 HTTP 状态和延迟。
- [ ] 自定义可信端点可保存、重启后仍存在，并可独立诊断。
- [ ] 无效端点、超时、限流、模型不可用分别显示可理解的分类提示。
- [ ] 删除凭据后应用返回首次设置界面。
- [ ] 日志、项目文件和普通设置文件中不出现 API Key 明文。

## 4. Agent 与权限边界

- [ ] `mimo-v2.5` 为默认模型，`mimo-v2.5-pro` 位于高级模型选择。
- [ ] 普通对话有真实流式输出，首段输出无明显整段等待。
- [ ] 只读模式不会修改文件。
- [ ] 工作区写入模式可修改当前项目，并对越界操作请求审批。
- [ ] 选择完全访问时必须先显示高风险确认；取消后仍保持原权限。
- [ ] 完全访问启用后，顶部持续显示醒目的权限标识。
- [ ] 命令、文件和权限审批显示原因、工作目录、授权边界与网络访问需求。
- [ ] 拒绝审批后任务不会执行被拒绝的操作。
- [ ] 停止任务后 Runtime 和子进程可结束，应用仍可继续提交新任务。
- [ ] 右侧“活动”面板实时显示请求、通知和响应，并可展开查看原始协议详情。

## 5. 项目、Git 与 Diff

- [ ] 可选择真实本地文件夹并在重启后恢复。
- [ ] 非 Git 文件夹显示明确状态，不阻塞普通对话。
- [ ] Git 分支、变更数量和工作区状态显示正确。
- [ ] 新增、修改、删除、已暂存和未暂存文件都能自动刷新。
- [ ] 右侧可按文件选择并审阅 Diff。
- [ ] Agent 完成任务后右侧 Git 状态会自动更新。

## 6. 线程与恢复

- [ ] 可创建新线程并在不同线程间切换。
- [ ] 重启应用后线程标题、消息、模型、权限和 Diff 可恢复。
- [ ] 恢复历史线程后可继续对话。
- [ ] 归档、恢复归档和删除线程行为正确。
- [ ] 旧 JSON 数据迁移到 SQLite 后内容完整，重复启动不会重复迁移。
- [ ] 异常退出后已落盘线程仍可恢复。
- [ ] 重启并恢复历史线程后，右侧“活动”面板仍显示已持久化审计记录。

## 7. 代表性任务

- [ ] A：询问项目结构并获得准确回答，不修改文件。
- [ ] B：定位一个失败测试，修改代码并运行验证。
- [ ] C：触发命令审批，分别验证允许一次、会话允许和拒绝。
- [ ] D：制造错误 Key、错误端点和 Provider 失败，确认结构化错误与恢复路径。

## 8. 证据与缺陷

每个失败项至少记录以下信息：

- 操作步骤和期望结果；
- 实际结果和截图；
- 应用版本、提交和 Actions 链接；
- 可脱敏的错误文本；
- 是否稳定复现，以及复现频率。

私测准入要求：第 2、3、4、5、6 节没有阻断级缺陷，代表性任务 A-D 全部可完成。
